## Syftet med uppgiften är att:
Syftet med den här uppgiften är att förstå versionshanteringssystem, att lära sig använda Git och att använda någon molntjänst för att plublicera Git-baserade datasamlingar.
## Svara på teorifrågor
I uppgiften så ska en del teorifrågor besvaras, vilket är ett bra sätt att söka upp och ta till sig information. Frågorna som ska besvaras är:

### Förklara följande begrepp:
1. Workspace
2. Staging area
3. Local repository
4. Remote repository
5. Branch
6. Merge

### Beskriv följande kommandon:
1. git config
2. git init
3. git status
4. git add
5. git commit
6. git push
7. git checkout
8. git pull
9. git merge
10. git fetch
11. git log

Beskriv även hur man går till väga för att ignorera vissa filer eller kataloger att indexeras med Git.

## Skapa konto och repo med README-fil
Skapa ett eget användarkonto på Github. Där ska man skapa ett remote repository och projekt på Github som består av en README fil.
README filen (som jag skriver i nu) ska skapas med markdown och ge en kort beskrivning av uppgiften.

### Skapa en personlig README-fil för min profil
Skapa en README-fil med passande innehåll för min profil. Information i denna ska då sedan visas upp om någon besöker min Github profil.
